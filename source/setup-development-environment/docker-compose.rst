نصب داکر کامپوز
=====================

Docker Compose ابزاری است که به کاربران اجازه می‌دهد تا چندین کانتینر Docker را با استفاده از یک فایل پیکربندی واحد، به نام `docker-compose.yml`, تعریف و مدیریت کنند. با استفاده از Docker Compose، می‌توان یک مجموعه از سرویس‌های تحت Docker را به راحتی استقرار داد، متوقف کرد و به طور هماهنگ مدیریت کرد. این ابزار به خصوص برای پروژه‌های چند بخشی (Multi-tier) بسیار مفید است، زیرا امکان تعریف وابستگی‌ها و مدیریت تمامی سرویس‌ها به صورت یکجا را فراهم می‌کند. به عنوان مثال، می‌توانید وب سرور، پایگاه داده، و سرویس‌های کش‌کردن را در یک فایل پیکربندی تعریف کرده و سپس با یک دستور ساده، تمامی این سرویس‌ها را به صورت هماهنگ اجرا کنید. Docker Compose باعث ساده‌تر شدن فرآیند توسعه، تست و استقرار برنامه‌های چند بخشی می‌شود.


Docker Compose، معمولاً با نسخه‌های جدید Docker همراه است. می‌توانید آن را با دستور زیر بررسی کنید:

  
.. code-block:: bash

    docker compose --version


گاهی ممکن است که شما از نسخه‌های قدیمی داکر استفاده می‌کنید، یا اینکه آن را از روی مخزن‌های پیش فرض Ubuntu 22.04 نصب کرده‌اید. در این حالت docker-compose به صورت یک ماژول نصب نخواهد شد. در این حالت می‌توانید دستور زیر را امتحان کنید:


.. code-block:: bash

    docker-compose --version
