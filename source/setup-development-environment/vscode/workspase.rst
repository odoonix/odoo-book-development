محیط کاری
=========================

در VS Code، محیط کاری یا Workspace مجموعه‌ای از فایل‌ها و پوشه‌هاست که به عنوان یک پروژه با هم کار می‌کنند. این محیط به شما این امکان را می‌دهد که تنظیمات خاصی را برای پروژه‌های خود تعیین کنید و دسترسی سریع به فایل‌های مهم داشته باشید. در حالت معمول، یک Workspace می‌تواند شامل یک یا چند پوشه باشد که شما آن‌ها را به پروژه خود اضافه کرده‌اید.

تنظیمات محیط کاری در فایل‌هایی مانند `settings.json` ذخیره می‌شوند که شامل پیکربندی‌های خاص برای هر پروژه است. این تنظیمات می‌توانند شامل مسیرهای اختصاصی برای اجرای کد، تنظیمات linting، و حتی دستورات خاصی باشند که باید به‌طور خودکار اجرا شوند. با استفاده از VS Code Workspaces، می‌توانید پروژه‌های خود را سازماندهی کرده و تنظیمات منحصر به فرد هر پروژه را به راحتی مدیریت کنید.

در VS Code، دو روش اصلی برای تعیین تنظیمات وجود دارد: استفاده از پوشه `.vscode` و استفاده از فایل `file.code-workspace`. هر کدام از این روش‌ها ویژگی‌ها و کاربردهای خاص خود را دارند. بیایید هر دو روش را به طور کامل بررسی کنیم.

استفاده از پوشه `.vscode`
---------------------------

در این روش، تنظیمات مرتبط با پروژه در فایل‌های جداگانه‌ای داخل پوشه‌ای به نام `.vscode` ذخیره می‌شوند. این پوشه باید در ریشه پروژه قرار گیرد. هر فایل تنظیمات در این پوشه وظایف خاصی دارد. این فایل‌ها شامل موارد زیر می‌شوند:

- **`settings.json`**: این فایل شامل تنظیمات کلی و سفارشی‌سازی‌های محیط کاری برای پروژه است. برای مثال، می‌توانید تنظیمات linting، قالب‌بندی کد، و مسیرهای خاص پروژه را در اینجا تعیین کنید.
- **`launch.json`**: این فایل پیکربندی‌های مربوط به اجرای کد و Debugging را شامل می‌شود. می‌توانید تنظیمات مربوط به نقاط قطع (breakpoints)، مسیر فایل اجرایی و آرگومان‌ها را در اینجا تعیین کنید.
- **`tasks.json`**: این فایل برای تعریف وظایف (tasks) اتوماتیک استفاده می‌شود. می‌توانید وظایف مختلفی مانند ساخت (build)، تست و اجرای اسکریپت‌های سفارشی را در این فایل تعریف کنید.

این روش مناسب است اگر شما بخواهید تنظیمات مختلف پروژه را در فایل‌های مجزا نگه دارید و به طور واضحی هر فایل مسئولیت خاص خود را داشته باشد.

استفاده از فایل `file.code-workspace`
----------------------------------

در این روش، تمام تنظیمات محیط کاری در یک فایل به نام `file.code-workspace` ذخیره می‌شوند. این فایل تمامی تنظیمات مرتبط با پروژه را به صورت یکجا شامل می‌شود. ساختار این فایل به صورت زیر است:

.. code-block::json

    {
        "folders": [
            {
                "path": "."
            }
        ],
        "settings": {
            // تنظیمات کلی پروژه
        },
        "launch": {
            // تنظیمات اجرای کد و Debugging
        },
        "tasks": {
            // تنظیمات وظایف (tasks)
        }
    }


در این روش، تمامی تنظیمات محیط کاری به صورت مجتمع در یک فایل واحد ذخیره می‌شوند. این روش مناسب است اگر شما بخواهید تمامی تنظیمات را در یک مکان متمرکز داشته باشید و دسترسی سریع‌تری به آن‌ها داشته باشید.

هر دو روش مزایا و معایب خاص خود را دارند و می‌توانید بر اساس نیازها و ترجیحات خود یکی از آن‌ها را انتخاب کنید. اگر نیاز به جداسازی و سازماندهی بیشتر تنظیمات دارید، استفاده از پوشه `.vscode` مناسب‌تر است. در غیر این صورت، استفاده از فایل `file.code-workspace` می‌تواند گزینه‌ای کارآمد باشد.


ایجاد و مدیریت محیط کاری
------------------------------

ما در تمام بخش‌های این کتاب این روش را استفاده می‌کنیم. این فایل به صورت خودکار در مسیر اصلی پروژه با دستور زیر ایجاد می‌شود. دستور زیر برای ایجاد یک محیط کاری (Workspace) در VS Code استفاده می‌شود:

.. code-block:: bash

    otoolbox \
        --odoo 18.0
        --path .
        workspace init


این دستور یک فایل به نام `18.0-file.code-workspace` در پوشه `.otoolbox` ایجاد می‌کند. فایل `18.0-file.code-workspace` شامل تمامی تنظیمات محیط کاری برای نسخه 18.0 Odoo است. این تنظیمات می‌توانند شامل مسیرهای مورد نیاز برای ماژول‌ها، تنظیمات linting، پیکربندی Debugging و سایر تنظیمات مربوط به پروژه باشند.

به این ترتیب، با اجرای این دستور و ایجاد فایل `18.0-file.code-workspace`، می‌توانید تمامی تنظیمات مورد نیاز برای پروژه Odoo خود را به صورت متمرکز و یکجا در اختیار داشته باشید و به راحتی از آن‌ها استفاده کنید. این روش به شما این امکان را می‌دهد که به سرعت محیط کاری خود را پیکربندی کرده و از امکانات و ابزارهای VS Code به بهترین شکل بهره‌برداری کنید.

برای مشاهده آخرین راهنمای مربوط به دستور `otoolbox workspace init` و تنظیم‌های مورد نیاز، می‌توانید از دستور زیر استفاده کنید:

.. code-block:: bash

    otoolbox workspace init --help


این دستور اطلاعات کاملی در مورد نحوه استفاده از `otoolbox` برای ایجاد یک Workspace فراهم می‌کند، شامل آرگومان‌ها و گزینه‌های مختلفی که می‌توانید برای پیکربندی بهتر محیط کاری خود استفاده کنید. با استفاده از این راهنما، می‌توانید تنظیمات مورد نیاز خود را به راحتی اعمال کنید و بهترین بهره‌برداری را از محیط توسعه خود داشته باشید.

پوشه‌ها
------------------


برای هر سازمان (organization) در GitHub، ما یک پوشه جداگانه در محیط کاری خود ایجاد می‌کنیم. این پوشه به عنوان محل ذخیره تمامی مخازن (repositories) مرتبط با آن سازمان عمل می‌کند. به عنوان مثال، اگر ما بخواهیم سازمان OCA را به پروژه خود اضافه کنیم، یک پوشه با نام OCA ایجاد می‌کنیم که در آن تمامی مخازن مرتبط با این سازمان قرار می‌گیرند.

در هر پوشه سازمان، یک لیست از پوشه‌های دیگر داریم که هرکدام نمایانگر یک مخزن GitHub هستند. هر مخزن شامل چندین افزونه Odoo می‌باشد. به عنوان نمونه، در پوشه OCA، یک مخزن به نام `iot` داریم که شامل افزونه‌های مختلف مرتبط با اینترنت اشیا (IoT) است. این ساختار به ما کمک می‌کند تا تمامی افزونه‌های Odoo را به صورت منظم و سازمان‌یافته نگهداری کنیم و به راحتی به هر کدام از آن‌ها دسترسی داشته باشیم.
