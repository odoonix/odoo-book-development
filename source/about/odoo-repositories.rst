مخزن‌های نرم‌افزاری اودوو
============================



Odoo همه کدهای خود را در GitHub میزبانی می کند و بستری برای گزارش باگ ها و مشکلات مربوط به نسخه های پایدار فراهم می کند. کاربران به مخزن کد دسترسی دارند، صرف نظر از عمومی یا خصوصی بودن آن. علاوه بر این، مشارکت‌کنندگان می‌توانند درخواست‌های کششی (PR) را برای پیشنهاد ویژگی‌های جدید ارسال کنند. روش ترجیحی ایجاد درخواست های کششی به جای فشار دادن مستقیم تغییرات به GitHub است. Odoo شامل چندین مخزن است که هر کدام عملکردهای خاصی را انجام می دهند. بیایید جزئیاتی در مورد مخازن مرکزی در اکوسیستم Odoo بررسی کنیم.

مخزن Odoo Git عمومی است و به عنوان نسخه جامعه Odoo عمل می کند. دسترسی به آن برای همه باز است و به توسعه دهندگان این امکان را می دهد تا به کدها برای اهداف توسعه خود مراجعه کنند.

مخزن Odoo Enterprise یک مخزن خصوصی است که منحصراً به نسخه سازمانی Odoo اختصاص داده شده است. دسترسی به این مخزن محدود و محدود به شرکای معتبر Odoo است.

مخزن Odoo Dev یک مخزن توسعه فعال است که برای دسترسی عموم باز است. این به عنوان فضایی برای تلاش‌های توسعه مداوم عمل می‌کند و به هر کسی اجازه می‌دهد به پایگاه کد مراجعه کرده و به آن کمک کند.

Odoo یک چرخه انتشار سالانه با نسخه های اصلی مانند 13.0، 14.0، 15.0، 16.0،18،0 و غیره را دنبال می کند. علاوه بر این، نسخه های کوچکی مانند saas-16.1، saas-16.2 و غیره وجود دارد که به طور خاص برای ارائه های SAAS طراحی شده اند. برای دسترسی به کد یک نسخه خاص، می توانید آن را از شاخه های مربوطه در مخزن دانلود کنید. این سیستم نسخه سازی ساختاریافته به کاربران این امکان را می دهد که نسخه های خاصی را بر اساس نیاز خود انتخاب کرده و با آنها کار کنند.
