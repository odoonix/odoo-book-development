pgadmin#

pgAdmin یک نرم‌افزار مدیریت پایگاه داده رایگان و متن‌باز است که برای مدیریت و توسعه‌ی PostgreSQL طراحی شده. این ابزار، محیط کاربری گرافیکی راحت و کاربرپسندی را برای انجام اعمال مختلف بر روی پایگاه‌های داده‌ی PostgreSQL فراهم می‌کند. با استفاده از pgAdmin، می‌توانید کوئری‌های SQL بنویسید و اجرا کنید، جداول و پابلیکیشن‌های مختلف را مدیریت کنید، و داده‌ها را به صورت گرافیکی مشاهده و تغییر دهید. همچنین این نرم‌افزار به شما امکان می‌دهد تا کاربران و نقش‌های مختلفی را برای پایگاه‌های داده خود تعریف کنید و سطوح دسترسی را مدیریت کنید.

این یک نمونه تغییرات است

pgAdmin از طریق رابط کاربری تحت وب و همچنین به صورت دسکتاپ در دسترس است و می‌تواند روی سیستم‌عامل‌های مختلفی مانند ویندوز، لینوکس و macOS اجرا شود. از ویژگی‌های برجسته‌ی این نرم‌افزار می‌توان به پشتیبانی از عملیات‌های پیشرفته‌ی پایگاه داده، نمایش بصری داده‌ها و ساختار پایگاه‌های داده به صورت درختی، و ارائه‌ی گزارش‌ها و آمارهای مختلف اشاره کرد. با استفاده از pgAdmin، توسعه‌دهندگان و مدیران پایگاه داده می‌توانند به راحتی و با کارآیی بالا پایگاه‌های داده‌ی خود را مدیریت و بهینه‌سازی کنند.

راه اندازی با داکر#

مطمئناً، می‌توانید pgAdmin را به راحتی با -----نمونه----- استفاده از Docker نصب کنید و به PostgreSQL 16 متصل شوید. فرض ما این است که PostgreSQL 16 از قبل روی Ubuntu 24.04 نصب و اجرا شده است. در اینجا، مراحل نصب و راه‌اندازی pgAdmin با Docker را توضیح می‌دهیم:

گام‌های نصب و راه‌اندازی pgAdmin با Docker

  1. ایجاد شبکه Docker (اختیاری ولی توصیه شده)

    ایجاد یک شبکه Docker برای ارتباط آسان بین کانتینرهای PostgreSQL و pgAdmin اولین کاری است که باید انجام شود. این کار با استفاده از دستور زیر انجام خواهد شد:

    docker network create odoo-network

    این شبکه به کانتینرها اجازه می‌دهد که به طور مستقیم با یکدیگر ارتباط برقرار کنند. البته در گام‌های بعد به شما نشان خواهیم داد که چطور کاینرها چطور به این شبکه متصل کنید.

  2. اجرای کانتینر pgAdmin 4

    اکنون می‌توانید دستور زیر را برای اجرای pgAdmin 4 در Docker اجرا کنید:

    docker run -d \
    --name pgadmin \
    -e PGADMIN_DEFAULT_EMAIL=your_email@domain.com \
    -e PGADMIN_DEFAULT_PASSWORD=your_password \
    --network odoo-network \
    -p 80:80 \
    dpage/pgadmin4

    در این دستور:

    • -d کانتینر را در پس‌زمینه اجرا می‌کند.

    • --name نام کانتینر را به عنوان pgadmin-container تعریف می‌کند.

    • -e PGADMIN_DEFAULT_EMAIL و -e PGADMIN_DEFAULT_PASSWORD ایمیل و رمز عبور پیش‌فرض برای ورود به pgAdmin 4 را تنظیم می‌کنند.

    • --network شبکه‌ای که کانتینر به آن متصل می‌شود، را تعیین می‌کند. این همان شبکه‌ای است که در مرحله قبلی ایجاد کردیم.

    • -p 80:80 پورت 80 روی میزبان را به پورت 80 کانتینر متصل می‌کند.

    • dpage/pgadmin4 تصویر Docker رسمی pgAdmin 4 را از Docker Hub دانلود و اجرا می‌کند.

  3. دسترسی به pgAdmin 4

    مرورگر وب خود را باز کنید و به آدرس http://localhost بروید. شما باید صفحه ورود pgAdmin 4 را مشاهده کنید. با استفاده از ایمیل و رمز عبوری که در مرحله قبل تنظیم کرده‌اید، وارد شوید.

  4. اتصال به PostgreSQL 16

    پس از ورود به محیط pgAdmin 4: - روی گزینه "Add New Server" کلیک کنید. - نام سرور را وارد کنید و به بخش "Connection" بروید. - در قسمت "Host name/address"، نام کانتینر PostgreSQL که قبلاً ایجاد کرده‌اید (در این مثال postgresql16) را وارد کنید. - پورت را به 5432 تنظیم کنید و اطلاعات کاربری PostgreSQL خود (username و password) را وارد کنید. - روی "Save" کلیک کنید تا سرور به pgAdmin 4 اضافه شود.

با انجام این مراحل، شما به راحتی می‌توانید pgAdmin 4 را با استفاده از Docker نصب کنید و به PostgreSQL 16 متصل شوید. اکنون می‌توانید از pgAdmin برای مدیریت پایگاه داده PostgreSQL خود به صورت گرافیکی استفاده کنید.

توجه

در بخش قبل اجرا کردن پایگاه داده با استفاده از داکر را بیان کردیم. اما در انجا در مورد اتصال به شبکه‌های مجاری حرفی به میان آورده نشد. برای اتصال بین این دو برنامه باید پعد از ایجاد شبکه، کانتینر ایجاد شده برای پایگاه داده را از بین برده و یک کانتینر جدید ایجاد کنید. کانتینر جدید باید به شبکه ایجاد شده متصل باشد. کد زیر نحوه انجام این کار را نشان می‌دهد

docker run -d \
    --name postgres16 \
    --network odoo-network \
    -e POSTGRES_USER=your_user \
    -e POSTGRES_PASSWORD=your_password \
    -e POSTGRES_DB=your_database \
    -v /home/your_user/postgresql_data:/var/lib/postgresql/data \
    -p 5432:5432 \
    postgres:16

راه اندازی با استفاده از داکر کامپوز#

بله، استفاده از Docker Compose می‌تواند فرآیند نصب و اجرای PostgreSQL 16 و pgAdmin 4 را ساده‌تر و هماهنگ‌تر کند. فرض بر این است که Docker از قبل نصب شده و شما می‌خواهید از Docker Compose برای مدیریت این کانتینرها استفاده کنید. در اینجا، مسیر انجام این کار را توضیح می‌دهم:

مراحل راه‌اندازی PostgreSQL 16 و pgAdmin 4 با Docker Compose

  1. ایجاد فایل `docker-compose.yml`

    ابتدا یک پوشه برای پروژه خود ایجاد کنید و به آن مسیر بروید. سپس یک فایل به نام docker-compose.yml در این پوشه ایجاد کنید:

    version: '3.8'

services:
    postgres:
        image: postgres:16
        container_name: postgresql16
        environment:
            POSTGRES_USER: your_user
            POSTGRES_PASSWORD: your_password
            POSTGRES_DB: your_database
        volumes:
            - /home/your_user/postgresql_data:/var/lib/postgresql/data
        ports:
            - "5432:5432"
        networks:
            - odoo-network

    pgadmin:
        image: dpage/pgadmin4
        container_name: pgadmin-container
        environment:
            PGADMIN_DEFAULT_EMAIL: your_email@domain.com
            PGADMIN_DEFAULT_PASSWORD: your_password
        ports:
            - "80:80"
        networks:
            - odoo-network

networks:
    odoo-network:
        driver: bridge

    در این فایل:

    • version مشخص کننده نسخه Docker Compose است.

    • services سرویس‌های مختلفی که می‌خواهید اجرا کنید را تعریف می‌کند: postgres و pgadmin.

    • postgres شامل تنظیمات مرتبط با PostgreSQL 16 است.

    • pgadmin شامل تنظیمات مرتبط با pgAdmin 4 است.

    • networks شبکه‌ای مشترک برای ارتباط بین کانتینرها را تعریف می‌کند.

    • تمامی مسیر‌ها و متغیر‌های محیطی را بر اساس نیاز خود تنظیم کنید.

  2. اجرای Docker Compose

    در ترمینال به مسیر پوشه‌ی پروژه‌ی خود بروید و دستور زیر را اجرا کنید:

    docker compose up -d

    این دستور Docker Compose را فعال می‌کند و سرویس‌های تعریف شده در فایل docker-compose.yml را در پس‌زمینه اجرا می‌کند.

  3. بررسی وضعیت سرویس‌ها

    برای بررسی وضعیت سرویس‌ها و کانتینرها می‌توانید از دستور زیر استفاده کنید:

    docker compose ps

    همچنین، برای مشاهده لاگ‌ها می‌توانید از دستور زیر استفاده کنید:

    docker compose logs

  4. دسترسی به pgAdmin 4

    مرورگر وب خود را باز کنید و به آدرس http://localhost بروید. شما باید صفحه ورود pgAdmin 4 را مشاهده کنید. با استفاده از ایمیل و رمز عبوری که قبلاً در فایل docker-compose.yml تنظیم کرده‌اید، وارد شوید.

  5. اتصال به PostgreSQL 16

    پس از ورود به محیط pgAdmin 4: - روی گزینه "Add New Server" کلیک کنید. - نام سرور را وارد کرده و به بخش "Connection" بروید. - در قسمت "Host name/address"، نام سرویس PostgreSQL که در فایل docker-compose.yml تعریف شده است (postgresql16) را وارد کنید. - پورت را به 5432 تنظیم کنید و اطلاعات کاربری PostgreSQL خود (username و password) را وارد کنید. - روی "Save" کلیک کنید تا سرور به pgAdmin 4 اضافه شود.

با انجام این مراحل، PostgreSQL 16 و pgAdmin 4 با استفاده از Docker Compose راه‌اندازی شده و داده‌ها در پوشه‌ی مونت شده‌ی محلی ذخیره می‌شوند. این روش به شما امکان می‌دهد به راحتی محیط توسعه خود را مدیریت و پایگاه داده خود را بهینه‌سازی کنید.

توجه

برای توقف سرویس‌ها، می‌توانید از دستور زیر استفاده کنید:

docker compose down

امیدوارم این راهنما به شما کمک کند! اگر سوال یا مشکلی داشتید، حتماً بپرسید.
