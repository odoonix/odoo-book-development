اجرای تست#

یکی از جنبه‌های کلیدی در توسعه نرم‌افزار اودوو، امکان اجرای تست‌های از پیش تعریف‌شده است که به توسعه‌دهندگان اجازه می‌دهد سیستم را به سبک‌های مدرن طراحی نرم‌افزار مانند TDD (Test-Driven Development) یا BDD (Behavior-Driven Development) توسعه دهند. این قابلیت نه‌تنها کیفیت کد را بهبود می‌بخشد، بلکه اطمینان می‌دهد که تغییرات اعمال‌شده در سیستم، عملکرد مورد انتظار را حفظ می‌کنند. در این بخش، به شما نشان می‌دهیم که چگونه می‌توانید با استفاده از تنظیمات Workspace Configuration در VS Code، تست‌های اودوو را به‌سادگی اجرا کنید.

در رویکرد TDD، توسعه‌دهندگان ابتدا تست‌ها را می‌نویسند و سپس کدی را پیاده‌سازی می‌کنند که این تست‌ها را با موفقیت پشت سر بگذارد. در BDD نیز تمرکز بر رفتار سیستم از دید کاربر نهایی است که از طریق سناریوهای تست تعریف می‌شود. اودوو این امکان را فراهم کرده است که تست‌های خودکار (مانند تست‌های واحد یا یکپارچه‌سازی) به‌راحتی اجرا شوند، و ما با استفاده از VS Code این فرآیند را یکپارچه‌تر کرده‌ایم.

مشابه با بخش قبلی که اجرای سرور اودوو را توضیح دادیم، در فایل تنظیمات Workspace پروژه، یک نمونه اجرایی (Launch Configuration) به بخش Launch اضافه شده است که به‌طور خاص برای اجرای تست‌های اودوو طراحی شده است. این پیکربندی شامل دستورات و پارامترهای لازم برای اجرای تست‌ها به‌صورت خودکار است و نیازی به استفاده از خط فرمان به‌صورت دستی ندارد.

برای اجرای تست‌های اودوو، مراحل زیر را دنبال کنید:

  1. باز کردن بخش Run and Debug: به نوار کناری VS Code بروید و روی آیکون Run and Debug کلیک کنید (آیکونی که معمولاً با یک مثلث و یک باگ نمایش داده می‌شود). این بخش دسترسی شما به تنظیمات اجرایی پروژه را فراهم می‌کند.

  2. انتخاب نام "Odoo Test": در منوی کشویی بالای پنجره Run and Debug، گزینه‌ای به نام "Odoo Test" را مشاهده خواهید کرد. این نام به نمونه اجرایی تعریف‌شده برای اجرای تست‌ها اشاره دارد. آن را انتخاب کنید.

  3. اجرای تست‌ها با کلید F5: پس از انتخاب "Odoo Test"، کلید F5 را فشار دهید. این کار دستورات لازم برای اجرای تست‌های اودوو را فعال می‌کند و نتایج تست‌ها را می‌توانید مستقیماً در ترمینال داخلی VS Code یا بخش خروجی مشاهده کنید. همچنین، در صورت وجود خطا، ابزارهای دیباگ VS Code به شما کمک می‌کنند تا مشکلات را شناسایی و رفع کنید.

این قابلیت در کنار ابزار Otoolbox که در بخش‌های قبلی معرفی شد، به‌طور کامل جریان کاری توسعه را تکمیل می‌کند. به عنوان مثال، پس از آماده‌سازی محیط توسعه با دستور otoolbox dev start و نوشتن یا به‌روزرسانی تست‌ها، می‌توانید به بخش Run and Debug بروید، "Odoo Test" را انتخاب کنید و با زدن F5، تست‌های خود را اجرا کنید. این فرآیند به شما امکان می‌دهد به‌سرعت بازخورد بگیرید و مطمئن شوید که کدتان با الزامات پروژه هم‌راستا است.

اجرای تست‌ها از طریق VS Code با این روش، چندین مزیت دارد: - سادگی و سرعت: نیازی به نوشتن دستورات پیچیده در ترمینال نیست؛ همه‌چیز با چند کلیک و یک میانبر قابل انجام است. - پشتیبانی از TDD و BDD: این روش توسعه‌دهندگان را تشویق می‌کند تا تست‌ها را به‌عنوان بخشی جدایی‌ناپذیر از فرآیند توسعه در نظر بگیرند. - رفع اشکال سریع: با استفاده از ابزارهای دیباگ داخلی VS Code، می‌توانید خطاهای تست‌ها را به‌سرعت شناسایی و برطرف کنید.

اضافه کردن نمونه اجرایی "Odoo Test" به تنظیمات Launch در VS Code، توسعه‌دهندگان اودوو را قادر می‌سازد تا تست‌های خود را به‌صورت مؤثر و کارآمد اجرا کنند. این قابلیت، همراه با ابزارهایی مثل Otoolbox، یک محیط توسعه حرفه‌ای و منسجم را فراهم می‌کند که هم بهره‌وری را افزایش می‌دهد و هم کیفیت نرم‌افزار نهایی را تضمین می‌کند. با دنبال کردن مراحل ساده بالا، می‌توانید به‌راحتی تست‌های خود را اجرا کرده و به سبک TDD یا BDD، سیستمی پایدار و قابل اعتماد بسازید.
