PostgreSQL
=============


PostgreSQL یک سیستم مدیریت پایگاه داده شی‌گرا و رابطه‌ای است که به عنوان یکی از پیشرفته‌ترین سامانه‌های مدیریت پایگاه داده متن‌باز شناخته می‌شود. این پایگاه داده به دلیل قابلیت‌های برجسته‌ای مانند پشتیبانی از تراکنش‌های ACID، انطباق با استاندارد SQL، امکان استفاده از انواع داده‌های پیچیده و اشیاء سفارشی، و قابلیت مدیریت حجم بالای داده‌ها، محبوبیت فراوانی دارد. PostgreSQL همچنین امکانات متنوعی مانند تکرار داده‌ها، افزونگی، و پشتیبان‌گیری پیشرفته را ارائه می‌دهد، که برای توسعه‌دهندگان و مدیران پایگاه داده‌ها، تجربه‌ای قدرتمند و قابل اطمینان فراهم می‌کند. این سیستم با جامعه‌ای فعال و پویا پشتیبانی می‌شود و به‌صورت منظم به‌روزرسانی‌ها و بهبودهایی دریافت می‌کند که امنیت، عملکرد و قابلیت‌های جدید را برای کاربران فراهم می‌سازد.

در سیستم مدیریت پایگاه داده اودوو، **کاربر پستگرس** یک نقش حیاتی ایفا می‌کند. اودوو برای ذخیره و مدیریت داده‌های خود از PostgreSQL استفاده می‌کند و هنگامی که اودوو نصب می‌شود، نیاز دارد تا یک کاربر پایگاه داده با سطح دسترسی مناسب تعریف شود. این کاربر، اغلب به نام `odoo` یا یک نام مشابه، باید توانایی ایجاد، خواندن، ویرایش و حذف داده‌ها در پایگاه داده را داشته باشد. نقش کاربر پستگرس در اودوو تنها به دسترسی داده‌ها محدود نمی‌شود؛ بلکه مسئول اجرای کوئری‌ها، کنترل تراکنش‌ها و حفظ یکپارچگی داده‌ها نیز می‌باشد. بنابراین، پیکربندی صحیح و امنیت این کاربر بسیار مهم است تا اودوو بتواند به صورت بهینه و امن عمل کند و از هرگونه اختلال و نفوذ پیشگیری نماید.

برای ساده‌تر کردن فرآیند نصب و راه‌اندازی PostgreSQL، از Docker استفاده می‌کنیم که یک روش مؤثر و بهینه است. Docker به ما امکان می‌دهد تا PostgreSQL را در یک کانتینر مجزا نصب و اجرا کنیم، بدون نیاز به پیکربندی دستی و پیچیده محیط‌ها. با استفاده از یک دستور ساده، می‌توانیم تصویر PostgreSQL رسمی را از داکر هاب دانلود کرده و یک کانتینر جدید ایجاد کنیم. این راه‌اندازی سریع و آسان به ما اجازه می‌دهد تا در کارهای توسعه و آزمایش تمرکز کنیم، زیرا Docker مدیریت وابستگی‌ها و تنظیمات محیطی را به عهده می‌گیرد. در ضمن، کانتینرهای Docker قابل حمل هستند، که به ما کمک می‌کند به راحتی محیط‌های توسعه مختلف را ایجاد و مدیریت کنیم.

راه اندازی با استفاده از داکر
----------------------------------------

فرض می‌کنیم Docker از قبل نصب شده است، و می‌خواهیم PostgreSQL 16 را راه‌اندازی کنیم و داده‌های ایجاد شده را در یک پوشه مونت شده ذخیره کنیم. در این آموزش، مراحل دقیق نصب و پیکربندی را توضیح می‌دهیم:

گام‌های راه‌اندازی PostgreSQL 16 با استفاده از Docker روی اوبونتو 22.04

1. **ایجاد پوشه برای ذخیره داده‌ها:**

    ابتدا یک پوشه در سیستم ایجاد کنید تا داده‌های PostgreSQL در آن ذخیره شود. این پوشه را به Docker کانتینر مونت خواهیم کرد.

    .. code-block:: bash

        mkdir -p /home/your_user/postgresql_data


    مسیر `/home/your_user/postgresql_data` را به هر مسیری که دوست دارید، تغییر دهید. تمام داده‌هایی که برای پایگاه داده‌ها در پایگاه داده ایجاد می‌شود، در این مسیر ذخیره خواهد شد. این پوشه را می‌توانید به عنوان یک بک آپ از پایگاه داده در نظر بگیرید.


2. **اجرا و کانفیگ کانتینر PostgreSQL:**

    یک کانتینر Docker با استفاده از تصویر PostgreSQL 16 اجرا کنید و پوشه داده‌ها را به آن مونت کنید.

    .. code-block:: bash

        docker run -d \
            --name postgres16 \
            -e POSTGRES_USER=your_user \
            -e POSTGRES_PASSWORD=your_password \
            -e POSTGRES_DB=your_database \
            -v /home/your_user/postgresql_data:/var/lib/postgresql/data \
            -p 5432:5432 \
            postgres:16


    در این دستور:

    - `-d` کانتینر را در پس‌زمینه اجرا می‌کند.
    - `--name` نام کانتینر را تعریف می‌کند، مانند `postgres16`.
    - متغیرهای محیطی `POSTGRES_USER`, `POSTGRES_PASSWORD` و `POSTGRES_DB` برای تعریف کاربر، رمز عبور و نام پایگاه داده استفاده می‌شوند. برای سادگی هر دو اینها را ما odoo در نظر می‌گیریم.
    - `-v /home/your_user/postgresql_data:/var/lib/postgresql/data` پوشه محلی را به پوشه داده‌های داخلی کانتینر مونت می‌کند.
    - `-p 5432:5432` پورت 5432 روی میزبان را به پورت 5432 کانتینر متصل می‌کند و امکان اتصال به PostgreSQL از خارج کانتینر را فراهم می‌کند.
    - `postgres:16` تصویر PostgreSQL 16 را که از Docker Hub دانلود می‌شود، تعیین می‌کند.

3. **بررسی کانتینر:**

    وضعیت کانتینر را بررسی کنید تا مطمئن شوید که صحیح اجرا شده است:

    .. code-block:: bash

        docker ps -a


    این دستور تمام کانتینرهای فعال را نشان می‌دهد. برای یافتن اطلاعات جزئی‌تر، می‌توانید از دستور زیر استفاده کنید:

    .. code-block:: bash

        docker logs postgres16

    برای وارد شدن به محیط کانتینر و اجرای دستورات PostgreSQL، از دستور زیر استفاده کنید:


    .. code-block:: bash

        docker exec -it db psql -U your_user -d your_database


    این دستور شما را به کنسول `psql` PostgreSQL می‌برد، جایی که می‌توانید کوئری‌ها و دستورات پایگاه داده را اجرا کنید.

**توجه:** مسیرهای محلی (`/home/your_user/postgresql_data`) و دیگر مقادیر مانند `POSTGRES_USER`, `POSTGRES_PASSWORD`, و `POSTGRES_DB` را با مقادیر دلخواه خود جایگزین کنید.


با انجام این مراحل، PostgreSQL 16 کاملاً پیکربندی شده و از داده‌ها در پوشه‌ی مونت شده‌ی محلی نگهداری خواهد شد. امیدوارم این راهنما مفید باشد! اگر نیاز به راهنمایی بیشتر دارید، خبرم کنید.


راه اندازی با داکر کامپوز
----------------------------------

بله، استفاده از Docker Compose برای راه‌اندازی PostgreSQL بسیار مفید و موثر است، زیرا به شما امکان می‌دهد تمام تنظیمات و کانتینرهای خود را در یک فایل YAML مدیریت کنید. در اینجا، نحوه انجام فرآیند راه‌اندازی PostgreSQL 16 با Docker Compose روی اوبونتو 22.04 را توضیح می‌دهم:

گام‌های راه‌اندازی PostgreSQL 16 با استفاده از Docker Compose

1. **ایجاد فایل `docker-compose.yml`**

    ابتدا یک پوشه برای پروژه خود ایجاد کنید و به آن مسیر بروید. سپس یک فایل به نام `docker-compose.yml` در این پوشه ایجاد کنید:

    .. code-block:: YAML

        version: '3.8'

        services:
            postgres16:
                image: postgres:16
                container_name: postgresql-container
                environment:
                    POSTGRES_USER: your_user
                    POSTGRES_PASSWORD: your_password
                    POSTGRES_DB: your_database
                volumes:
                    - /home/your_user/postgresql_data:/var/lib/postgresql/data
                ports:
                    - "5432:5432"


    در این فایل:
    - `version` مشخص کننده نسخه Docker Compose است.
    - `services` سرویس‌های مختلفی که می‌خواهید اجرا کنید را تعریف می‌کند. در این مورد، تنها یک سرویس PostgreSQL داریم.
    - `image` تصویر Docker که استفاده می‌کنید را مشخص می‌کند، در اینجا `postgres:16`.
    - `container_name` نام کانتینری که می‌خواهید داشته باشید را تعیین می‌کند.
    - `environment` متغیرهای محیطی مانند کاربر، رمز عبور، و پایگاه داده را تنظیم می‌کند.
    - `volumes` پوشه محلی را به مسیر داده‌های داخلی کانتینر مونت می‌کند.
    - `ports` پورت 5432 روی میزبان را به پورت 5432 کانتینر متصل می‌کند.

    این فایل ایجاد شده را در فصل‌های بعدی تکمیل خواهیم کرد تا یک سیستم کامل برای ایجاد محیط توسعه اودوو داشته باشیم.

2. **اجرای Docker Compose**

    در ترمینال به مسیر پوشه‌ی پروژه‌ی خود بروید و دستور زیر را اجرا کنید:

    .. code-block:: bash

        docker compose up -d


    این دستورDocker Compose را فعال می‌کند و سرویس‌های تعریف شده در فایل `docker-compose.yml` را در پس‌زمینه اجرا می‌کند.

3. **بررسی وضعیت سرویس‌ها**

    برای بررسی وضعیت سرویس‌ها و کانتینرها می‌توانید از دستور زیر استفاده کنید:

    .. code-block:: bash

        docker compose ps

    همچنین، برای مشاهده لاگ‌ها می‌توانید از دستور زیر استفاده کنید:

    .. code-block:: bash

        docker-compose logs

4. **اجرای دستورات PostgreSQL**

    برای ورود به محیط PostgreSQL داخل کانتینر و اجرای دستورات، از دستور زیر استفاده کنید:

    .. code-block:: bash

        docker compose exec postgres16 psql -U your_user -d your_database

    این دستور شما را به کنسول `psql` PostgreSQL می‌برد، جایی که می‌توانید کوئری‌ها و دستورات پایگاه داده را اجرا کنید.

با انجام این مراحل، PostgreSQL 16 با استفاده از Docker Compose راه‌اندازی شده و داده‌ها در پوشه‌ی مونت شده‌ی محلی ذخیره می‌شوند. این راه‌اندازی به شما امکان می‌دهد به راحتی محیط توسعه خود را مدیریت و پایگاه داده خود را بهینه‌سازی کنید.

**نکات کلیدی:**

- اطمینان حاصل کنید که مسیر‌ها و متغیر‌های محیطی را متناسب با نیازهای خود تغییر دهید.
- برای توقف سرویس‌ها، می‌توانید از دستور زیر استفاده کنید:

    .. code-block:: bash

        docker-compose down
