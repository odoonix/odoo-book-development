git
=============


Git یک سیستم کنترل نسخه توزیع‌شده است که به توسعه‌دهندگان نرم‌افزار امکان می‌دهد تغییرات کد منبع خود را پیگیری و مدیریت کنند. این ابزار قدرتمند به تیم‌های توسعه اجازه می‌دهد تا به صورت هماهنگ و موثر بر روی پروژه‌های بزرگ و پیچیده کار کنند. کاربردهای کلیدی Git شامل موارد زیر است:


1. **کنترل نسخه**: Git تمامی نسخه‌های مختلف کد منبع را ذخیره و پیگیری می‌کند. این قابلیت به توسعه‌دهندگان اجازه می‌دهد هر نسخه از کد را به آسانی بازیابی کنند و تغییرات بین نسخه‌های مختلف را مقایسه کنند.

2. **مدیریت شاخه‌ها (Branching)**: Git به کاربران امکان می‌دهد شاخه‌های جداگانه از کد منبع ایجاد کنند تا ویژگی‌های جدید را توسعه دهند یا باگ‌ها را بدون اختلال در شاخه اصلی (main) رفع کنند. این ویژگی به تیم‌ها اجازه می‌دهد تا چندین جریان کاری موازی را مدیریت کنند.

3. **ادغام کد (Merging)**: پس از توسعه و آزمایش یک ویژگی جدید در یک شاخه جداگانه، Git به شما امکان می‌دهد تا تغییرات را به شاخه اصلی ادغام کنید. این قابلیت به از بین بردن تضادها و تداخل‌ها کمک می‌کند و تضمین می‌کند که کد همیشه در حالت همگام و یکپارچه باقی می‌ماند.

4. **تاریخچه تغییرات**: Git یک تاریخچه جامع از تمامی تغییرات انجام شده در کد فراهم می‌کند. این قابلیت به تیم‌ها اجازه می‌دهد به راحتی متوجه شوند کدام تغییرات توسط کدام عضو تیم انجام شده و دلیل آن‌ها را مرور کنند.

5. **همکاری تیمی**: Git با ابزارهایی مانند GitHub، GitLab، و Bitbucket ادغام می‌شود و به تیم‌های توسعه‌دهنده امکان می‌دهد به صورت همزمان و از راه دور بر روی پروژه‌ها کار کنند. این پلتفرم‌ها ویژگی‌هایی مانند مدیریت درخواست‌های قطع (pull requests)، بررسی کد، و ادغام پیوسته (CI/CD) را فراهم می‌کنند.

گیت یکی از پر کاربردترین نرم افزارهایی است که هر بنرامه نویش در طول روز کاری خود با ان سرو کار دارد. از کاربردهای عملی این نرم افزار به موارد زیر می‌توان اشاره کرد.

- **توسعه نرم‌افزارهای بزرگ**: تیم‌های بزرگی که بر روی پروژه‌های پیچیده کار می‌کنند، می‌توانند از Git برای مدیریت تغییرات کدهای خود استفاده کنند و تقسیم‌بندی وظایف را به شکل موثری انجام دهند.
- **ردیابی باگ‌ها و رفع آن‌ها**: با پیگیری تغییرات کد، توسعه‌دهندگان می‌توانند به راحتی باگ‌های موجود را شناسایی کنند و رفع کنند، بدون اینکه نگرانی از دست دادن تغییرات جدیدتر داشته باشند.
- **استفاده در پروژه‌های متن‌باز**: جوامع متن‌باز از Git برای مشارکت و همکاری در پروژه‌های مختلف استفاده می‌کنند. GitHub و GitLab نمونه‌های محبوبی از پلتفرم‌های مبتنی بر Git هستند که جوامع توسعه‌دهنده را به هم می‌پیوندند.


از مزیت‌هایی که این نرم افزار نسبت به سایر نرم افزارهای مشابه دارد می‌توان به موارد زیر اشاره کرد:

- **توابع توزیع‌شده**: برخلاف سیستم‌های مدیریت نسخه مرکزی، Git به هر توسعه‌دهنده یک کپی کامل از مخزن (repository) پروژه را می‌دهد که امکان کار آفلاین را نیز فراهم می‌کند.
- **سرعت و کارآیی**: Git برای انجام عملیات‌ها سریع و بهینه طراحی شده است، که به توسعه‌دهندگان کمک می‌کند کار خود را با سرعت بیشتری انجام دهند.
- **امنیت و انعطاف‌پذیری**: با امکانات پیشرفته‌ای مانند امضاهای دیجیتال و سیستم‌های تأیید هویت، Git امنیت بالایی در مدیریت کد فراهم می‌کند.

Git به عنوان یکی از ابزارهای اساسی در توسعه نرم‌افزار، به تیم‌ها و توسعه‌دهندگان امکان می‌دهد کدهای خود را با دقت و کارآیی بالاتری مدیریت کنند و فرآیند توسعه را بهبود بخشند.

تاریخچه
-------------

جیث (Git) یک سیستم کنترل نسخه توزیع‌شده است که در سال 2005 توسط لینوس توروالدز، خالق لینوکس، توسعه داده شد. نیاز به یک سیستم کنترل نسخه جدید زمانی پیش آمد که توسعه‌دهندگان هسته لینوکس در استفاده از BitKeeper دچار محدودیت‌هایی شدند. BitKeeper یک سیستم کنترل نسخه‌ی تجاری بود که به عنوان تخت نظر فعالیت‌های توسعه هسته لینوکس مورد استفاده قرار می‌گرفت. اما بعد از ایجاد محدودیت‌ها در استفاده رایگان از آن، لینوس توروالدز تصمیم گرفت یک ابزار جدید و متن‌باز برای کنترل نسخه‌ی پروژه‌ها بسازد.

..
    ### تولد Git

لینوس طی مدت زمان کوتاهی از مارس تا آوریل 2005، به توسعه Git پرداخت تا خرابکاری‌ها و تضادهای مربوط به شاخه‌ها و نسخه‌ها در BitKeeper جبران شود. هدف اصلی Git این بود که یک سیستم کنترل نسخه سریع، غیر متمرکز و با قابلیت‌های پیشرفته برای مدیریت بزرگ‌ترین پروژه‌های نرم‌افزاری ایجاد کند.

..
    ### ویژگی‌های کلیدی

Git با بهره‌گیری از ویژگی‌هایی مانند ساختار گرافی از نسخه‌ها (DAG) و الگوریتم‌های هش رمزنگاری (SHA-1)، امکان ردیابی تغییرات با دقت و امنیت بالا را فراهم کرد. همچنین، Git به توسعه‌دهندگان اجازه می‌دهد به صورت آفلاین تغییراتی انجام دهند و از نسخه کپی محلی خود استفاده کنند، و این ویژگی توزیع‌شده بودن Git را برجسته می‌کند.

..
    ### رشد و پذیرش

از زمان انتشار اولیه، Git به سرعت توسط جوامع توسعه‌دهنده پذیرفته شد. در سال 2008، GitHub راه‌اندازی شد، که یک پلتفرم همکاری مبتنی بر Git بود و جوامع متن‌باز را به خود جذب کرد. GitHub ابزارها و خدماتی مانند هاستینگ پروژه، مدیریت درخواست‌های تغییر (Pull Request)، و مرور کد فراهم کرد که کار با Git را آسان‌تر و مؤثرتر کرد.

..
    ### توسعه‌های جدید

با گذر زمان، Git به نسخه‌های متعددی به‌روزرسانی شد، و با ویژگی‌های بیشتری مانند پشتیبانی از پروتکل‌های انتقال امن (SSH)، ادغام پیشرفته شاخه‌ها و قابلیت‌های تعامل بهتر با سایر ابزارهای کنترل نسخه بهبود یافت. امروز، Git به عنوان استاندارد طلایی برای مدیریت و ردیابی کد پروژه‌های نرم‌افزاری در سطح جهانی شناخته می‌شود.

..
    ### جایگاه کنونی

اکنون، Git نه‌تنها بخش جدایی‌ناپذیری از پروژه‌های متن‌باز است، بلکه در شرکت‌ها، سازمان‌ها و پروژه‌های تجاری نیز به صورت گسترده مورد استفاده قرار می‌گیرد. ابزارها و پلتفرم‌هایی مانند GitLab، Bitbucket، و Azure DevOps نیز با Git سازگارند و از کنترل نسخه آن بهره‌برداری می‌کنند.

تاریخچه‌ای که Git پشت سر گذاشته، آن را به یکی از قدرتمندترین و محبوب‌ترین ابزارهای مدیریت نسخه در دنیا تبدیل کرده است، که همچنان در حال بهبود و توسعه است تا نیازهای روزافزون توسعه‌دهندگان را پاسخ دهد.


نصب و راه اندازی
-------------------

برای نصب Git روی اوبونتو 22.04، مراحل زیر را دنبال کنید:

۱. به‌روزرسانی مخازن سیستم

    ابتدا مخازن APT سیستم خود را به‌روز کنید تا مطمئن شوید که بسته‌های موجود به‌روز هستند:

    .. code-block:: bash

        sudo apt update
        sudo apt upgrade -y


۲. نصب Git

    با استفاده از دستور زیر، Git را از مخازن پیش‌فرض اوبونتو نصب کنید:

    .. code-block:: bash

        sudo apt install git -y


۳. تأیید نصب Git

    پس از نصب، می‌توانید نسخه‌ی نصب‌شده را بررسی کنید تا مطمئن شوید که Git به‌درستی نصب شده است:

    .. code-block:: bash

        git --version


۴. تنظیمات اولیه Git

    برای استفاده‌ی بهتر از Git، نام کاربری و ایمیل خود را تنظیم کنید. این اطلاعات برای ثبت تاریخچه‌ی تغییرات و کدهای شما استفاده می‌شود:

    .. code-block:: bash

        git config --global user.name "Your Name"
        git config --global user.email "youremail@example.com"


۵. بررسی تنظیمات Git

    برای اطمینان از اعمال صحیح تنظیمات، می‌توانید تنظیمات جاری را با دستور زیر بررسی کنید:

    .. code-block:: bash

        git config --list


دستورات پر کاربرد
----------------------


حتماً! Git شامل دستورات بسیاری است که برای کنترل نسخه و مدیریت پروژه‌های نرم‌افزاری استفاده می‌شود. در اینجا، تعدادی از پرکاربردترین دستورات Git را توضیح می‌دهم و کاربرد هر کدام را بیان می‌کنم:


1. `git init`

این دستور برای ایجاد یک مخزن (repository) جدید Git استفاده می‌شود. وقتی در پوشه پروژه خود هستید و می‌خواهید کنترل نسخه را شروع کنید، می‌توانید این دستور را اجرا کنید:

.. code-block:: bash

    git init


2. `git clone`

برای کپی کردن یک مخزن موجود از سرور یا سیستم دیگری به سیستم خود، از این دستور استفاده می‌شود. این دستور تمام تاریخچه و فایل‌های مخزن را کپی می‌کند:

.. code-block:: bash

    git clone <repository_url>


3. `git add`

برای اضافه کردن تغییرات فایل‌ها به منطقه موقتی (staging area) استفاده می‌شود. این دستور می‌تواند برای اضافه کردن یک فایل خاص یا تمام فایل‌ها در پوشه جاری استفاده شود:


.. code-block:: bash

    git add <filename>
    git add .


4. `git commit`

برای ثبت تغییرات ذخیره‌شده در منطقه موقتی به مخزن محلی استفاده می‌شود. هر بار که تغییرات مهمی ایجاد می‌کنید، می‌توانید این دستور را با یک پیام توضیحی اجرا کنید:


.. code-block:: bash

    git commit -m "Commit message"


5. `git status`

این دستور وضعیت جاری مخزن را نمایش می‌دهد. شامل فایل‌های تغییر یافته، فایل‌های جدید اضافه‌شده به منطقه موقتی، و فایل‌هایی که هنوز اضافه نشده‌اند:


.. code-block:: bash

    git status


6. `git log`

این دستور تاریخچه تمام تعهدات (commits) در مخزن را نمایش می‌دهد. می‌توانید با استفاده از این دستور، تغییرات ثبت‌شده را مرور کنید:


.. code-block:: bash

    git log


7. `git pull`

برای بروزرسانی نسخه محلی مخزن با تغییرات از مخزن راه دور استفاده می‌شود. این دستور تغییرات جدید را از سرور دریافت و با مخزن محلی ادغام می‌کند:


.. code-block:: bash

    git pull


8. `git push`

برای ارسال تغییرات تعهدی محلی به مخزن راه دور استفاده می‌شود. پس از انجام تعهدات محلی، می‌توانید این دستور را اجرا کنید تا تغییرات شما به سرور ارسال شود:


.. code-block:: bash

    git push <remote_name> <branch_name>


9. `git branch`

این دستور برای ایجاد، نمایش یا حذف شاخه‌ها (branches) استفاده می‌شود. به عنوان مثال، برای ایجاد یک شاخه جدید:


.. code-block:: bash

    git branch <branch_name>


10. `git checkout`

برای سوئیچ به شاخه‌ای دیگر یا بازگرداندن فایل‌ها به نسخه قبلی استفاده می‌شود. همچنین می‌توان مخزن را به وضعیت قبلی برگرداند:


.. code-block:: bash

    git checkout <branch_name>


11. `git merge`

برای ادغام تغییرات از یک شاخه دیگر به شاخه جاری استفاده می‌شود. این دستور معمولاً برای ترکیب تغییرات ویژگی‌های جدید یا رفع باگ‌ها به شاخه اصلی استفاده می‌شود:


.. code-block:: bash

    git merge <branch_name>


12. `git remote`

برای مدیریت مخازن راه دور (remote repositories) استفاده می‌شود. این دستور به شما امکان می‌دهد مخازن راه دور را اضافه، حذف یا مشاهده کنید:


.. code-block:: bash

    git remote add <name> <url>
    git remote -v


- **کار با Git نیازمند تمرین و تجربه است تا به طور کامل با قابلیت‌ها و کاربردهای مختلف آن آشنا شوید.**
- **حتماً قبل از اجرای دستورات خاص، از وضعیت مخزن خود مطلع شوید تا از هرگونه تغییر ناخواسته جلوگیری شود.**
